# 제품 요구사항 정의서 (PRD)
## Git-CMS: Git 기반 하이브리드 문서화 시스템

### 1. 개요 (Executive Summary)
Git-CMS는 기술 개발자와 비기술 이해관계자 간의 간극을 좁히기 위해 설계된 이중 인터페이스 문서화 플랫폼입니다. 이 시스템은 **Git**을 단일 진실 공급원(Source of Truth)으로 활용하여, 개발자는 선호하는 환경(VS Code)에서 작업하고, 제품 관리자나 일반 사용자는 사용자 친화적인 웹 인터페이스를 활용하는 "하이브리드" 워크플로우를 가능하게 합니다. 이 시스템은 매끄러운 동기화 루프를 생성하여 문서가 항상 버전 관리되고, 투명하며, 접근 가능하도록 보장합니다.

### 2. 전략적 목표 (Strategic Goals)
- **통합된 진실 (Unified Truth):** 코드와 문서를 같은 곳에 위치시키거나(또는 Git을 통해 동기화하여) 코드와 문서 간의 "괴리(drift)"를 제거합니다.
- **AI 에이전트 친화적 (Agent-Ready):** AI 에이전트(Cline/Antigravity 등)가 읽고, 이해하고, 자율적으로 업데이트하기에 최적화된 구조적이고 결정론적인 파일 시스템 레이아웃을 제공합니다.
- **학습 곡선 제로 (Web):** 웹 인터페이스는 복잡한 Git 명령어(pull, push, merge)를 단순한 "저장(Save)" 및 "동기화(Sync)" 작업으로 추상화해야 합니다.

### 3. 상세 사용자 워크플로우 (Detailed User Workflows)

#### 3.1 개발자 워크플로우 (VS Code)
1.  **편집:** 개발자가 VS Code에서 `docs/architecture.md`를 엽니다.
2.  **커밋:** 기존 Git 도구를 사용하여 변경 사항을 커밋합니다.
3.  **푸시:** 중앙 저장소(Repository)로 푸시합니다.
    *   *결과:* 웹 인터페이스는 다음 빌드/새로고침 시 변경 사항을 자동으로 반영합니다.

#### 3.2 웹 사용자 워크플로우 (Browser)
1.  **탐색:** 사이드바를 통해 문서 트리를 탐색합니다.
2.  **편집:** "페이지 편집(Edit Page)"을 클릭합니다. 기능적인 마크다운 에디터가 로드됩니다.
3.  **저장:** "변경 사항 저장(Save Changes)"을 클릭합니다.
    *   *시스템 동작:* 시스템이 체계적인 Git 커밋을 생성합니다 (예: `docs: update architecture.md via web-ui`).
    *   *시스템 동작:* 원격 저장소로 자동 푸시합니다.

#### 3.3 충돌 시나리오 (Conflict Scenario)
웹 사용자가 파일을 열어둔 상태에서 서버/원격지의 파일이 변경된 경우 저장하려 할 때:
-   **요구사항:** 시스템은 충돌을 감지해야 합니다.
-   **해결:** 사용자에게 알림: "다른 사용자에 의해 파일이 변경되었습니다."
    -   *옵션 A:* 덮어쓰기 (강제 푸시와 동일 - *권장하지 않음*).
    -   *옵션 B:* 변경 사항 보기 (Diff 보기).
    -   *옵션 C:* 새로운 사본으로 저장.

### 4. 기능 요구사항 (Functional Requirements)

#### 4.1 콘텐츠 관리
-   **풍부한 마크다운 지원:** GFM (GitHub Flavored Markdown), 표, Mermaid 다이어그램, 콜아웃/알림.
-   **자산 관리:** 웹 UI에서 이미지 드래그 앤 드롭 업로드 (`docs/images/`에 저장).
-   **프론트매터:** 메타데이터(제목, 작성자, 태그, 최종 수정일)를 위한 YAML 프론트매터 지원.

#### 4.2 버전 관리 기능
-   **히스토리 보기:** 모든 페이지에 대해 해당 파일에 영향을 준 커밋 목록을 보여주는 "히스토리" 탭 제공.
-   **블레임/기여 (Blame/Attribution):** 섹션을 마지막으로 수정한 사람이 누구인지 시각적 표시.
-   **롤백:** (향후) 웹 UI에서 파일을 이전 버전으로 되돌리는 기능.

#### 4.3 AI 통합
-   **시맨틱 검색:** (향후) 에이전트가 관련 문맥을 찾을 수 있도록 벡터 기반 검색 지원.
-   **API 접근:** 에이전트가 UI 상호작용 없이 모든 문서를 나열하고, 내용을 읽고, 패치를 제출할 수 있는 명확한 엔드포인트 제공.

### 5. 비기능 요구사항 (Non-Functional Requirements)
-   **지연 시간:** 파일 저장 작업은 2초 이내에 완료되어야 합니다 (상류 Git 네트워크 지연 시간 제외).
-   **신뢰성:** 데이터 손상을 방지하기 위한 원자적(Atomic) 파일 쓰기.
-   **보안:** 웹 UI "쓰기" 작업에는 인증이 필요합니다. (MVP: 로컬용은 Basic Auth 또는 인증 없음, 배포용은 OIDC).

### 6. 로드맵 (Roadmap)
-   **1단계 (MVP):** 로컬 Next.js 앱, 로컬 Git 저장소, 기본 마크다운 읽기/쓰기.
-   **2단계:** 원격 Git 동기화, 충돌 감지, 이미지 업로드.
-   **3단계:** AI 에이전트 "관리인" (깨진 링크 자동 수정, 오래된 문서 업데이트).
